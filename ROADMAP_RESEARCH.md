# Craft Docs User Feedback Research - Roadmap Insights

Research conducted on: 2025-11-29
Focus areas: Daily Notes & Task Management

## Executive Summary

Based on Reddit discussions and user reviews, Craft Docs users appreciate the app's beautiful design and document creation capabilities, but consistently identify gaps in task management functionality and daily notes workflows. The community indicates that Craft is transitioning from a pure document editor to a more comprehensive productivity tool, but several key features are still missing.

## üéØ Most Common Feature Requests

### Task Management & GTD

1. **Kanban View for Tasks** ‚≠ê‚≠ê‚≠ê
   - Users explicitly want Kanban boards for visual task management
   - Currently missing from Craft's official feature set
   - Critical for project management workflows

2. **Robust Task/Todo System** ‚≠ê‚≠ê‚≠ê
   - Users report that Craft is "not a task manager" and lacks comprehensive GTD integration
   - Many users resort to combining Craft with dedicated task managers like Things 3
   - Need for better task integration beyond basic checkboxes

3. **Reminders & Due Dates** ‚≠ê‚≠ê‚≠ê
   - Missing to-dos with reminders
   - No native reminder system for time-sensitive tasks
   - Users want calendar integration for tasks

4. **GTD/Project Management Workflow** ‚≠ê‚≠ê
   - Craft 3.0 introduced "Inbox" inspired by GTD framework
   - However, users report "no GTD or project management workflow baked into the task integration"
   - Need for more structured productivity methodologies

### Daily Notes Improvements

5. **Better Search Within Daily Notes** ‚≠ê‚≠ê‚≠ê
   - Search functionality is "very rudimentary"
   - Doesn't find content within files or match between blocks
   - Can't effectively search between title and content
   - Critical for users managing extensive daily note archives

6. **Enhanced Linking & References** ‚≠ê‚≠ê
   - Users want improved bidirectional links
   - Content referencing capabilities
   - Auto-linking features
   - Custom templates for daily notes

7. **Tagging System** ‚≠ê‚≠ê
   - Tags have been "long-requested by the Craft community"
   - Currently missing despite being a standard feature in competing apps
   - Would enhance daily note organization and retrieval

### Database & Structure

8. **Database Support** ‚≠ê‚≠ê‚≠ê
   - No database/table functionality with formulas
   - Tables don't support calculations
   - Users coming from Notion specifically miss this

9. **OCR for Images** ‚≠ê‚≠ê
   - No ability to search text within images
   - Limiting for users who embed screenshots in daily notes

### Organization & Sharing

10. **Folder Sharing** ‚≠ê‚≠ê
    - Can only share individual documents, not folders
    - Limits team collaboration workflows

11. **Custom URLs** ‚≠ê‚≠ê
    - Missing custom URL support for documents

## üò§ Most Common Gripes & Pain Points

### User Interface & Experience

1. **UI Overcrowding** ‚≠ê‚≠ê‚≠ê
   - "Endless different views, confusing navigation and animations"
   - "Cramming every feature in the UI"
   - "Hidden elements behind multi-slide navs"
   - Users feel the app prioritizes adding features over focused note-taking experience
   - **Quote**: "Product seems to prioritize adding every single imaginable feature rather than just focusing on how users actually write notes"

2. **Font Size Issues (iOS)** ‚≠ê‚≠ê
   - Fonts too small on iOS
   - Cannot be adjusted
   - Accessibility concern

### Technical Issues

3. **Code Block Problems** ‚≠ê‚≠ê
   - Copying code (especially backward highlighting) gives line numbers instead of code
   - No keyboard shortcut for inserting formula blocks
   - Limits developer/technical user adoption

4. **Missing Basic Features** ‚≠ê‚≠ê
   - No voice typing
   - Limited template options (described as "old and repeated")
   - No multi-factor authentication
   - No public API

### Task Management Limitations

5. **Insufficient for GTD** ‚≠ê‚≠ê‚≠ê
   - "Craft was not created to be project management software"
   - "Implementing a GTD system in Craft wasn't better than alternatives like Notion"
   - Experts advise "against managing tasks in Craft"
   - Forces users to maintain separate task management tools

6. **No Mind Map Feature** ‚≠ê
   - Requested for visual thinking and planning

## üîç Daily Notes Specific Insights

### What Users Love
- Quick note feature that navigates to daily notes from any page
- Calendar view showing all days with daily notes
- Automated daily note creation
- Clean writing experience
- Integration with interstitial journaling workflows

### What Users Want Improved
1. **Search**: Better full-text search across all daily notes
2. **Linking**: Stronger bidirectional linking between daily notes
3. **Templates**: More flexible and modern daily note templates
4. **Task Integration**: Seamless task tracking within daily notes
5. **Quick Capture**: Faster inbox-to-daily-note workflows

---

## üìä Task Management Specific Insights

### Current State
- Craft 3.0 introduced Collections and Task Management features
- Inbox feature for quick task capture
- Basic task checkboxes in documents

### User Reality
- **Hybrid Approach**: Users combine Craft with dedicated task managers
  - "Daily task management living in Things while related notes for projects and tasks live in Craft"
- **Not Sufficient Alone**: Cannot replace dedicated task management tools
- **Missing GTD Integration**: No clear methodology support

### Critical Gaps
1. No Kanban/board views
2. No task dependencies or relationships
3. Limited task metadata (priority, tags, projects)
4. No recurring tasks
5. Poor task search and filtering
6. No task-specific reminders or notifications

## üí° Roadmap Recommendations for iOS App

### High Priority (Address First)

1. **Enhanced Task Management System**
   - Implement Kanban board view for tasks
   - Add task metadata: priority, tags, due dates, reminders
   - Create GTD-compliant workflow (Inbox ‚Üí Next Actions ‚Üí Projects ‚Üí Waiting)
   - Enable task filtering and smart lists

2. **Powerful Search Overhaul**
   - Full-text search across all content types
   - Search within daily notes archives
   - Filter by date ranges, tags, document types
   - Search preview with context

3. **Tagging System**
   - Document-level and block-level tags
   - Tag hierarchy and organization
   - Tag-based filtering and search
   - Tag autocomplete

### Medium Priority

4. **Better Daily Notes Integration**
   - Custom daily note templates
   - Quick capture to daily notes from anywhere
   - Daily note linking and referencing
   - Weekly/monthly rollup views

5. **Database Tables**
   - Tables with formulas and calculations
   - Sortable and filterable table views
   - Table templates

6. **UI Simplification**
   - Reduce navigation complexity
   - Focus modes for distraction-free writing
   - Customizable toolbar and navigation
   - Better discoverability without overwhelming UI

### Lower Priority

7. **Advanced Features**
   - Public API for integrations
   - Folder sharing capabilities
   - OCR for image text search
   - Mind mapping tools
   - Voice typing support

## üìà Success Metrics to Track

If implementing these features:

1. **Task Management Adoption**: % of users who use Craft tasks as their primary task manager
2. **Daily Notes Engagement**: Daily active users in daily notes section
3. **Search Usage**: Search queries per user, search success rate
4. **Tag Adoption**: % of documents with tags, tags per document
5. **User Retention**: Reduced churn to Notion/Obsidian alternatives
6. **NPS Score**: Improvement in user satisfaction, particularly for task management

## üéØ Positioning Strategy

Based on user feedback, consider positioning Craft as:

**Option A - "The Beautiful Note-Taking App"**
- Lean into design excellence
- Focus on writers and document creators
- Accept that users will use separate task managers
- Risk: Losing users to more comprehensive tools

**Option B - "The Complete Productivity Platform"** ‚≠ê (Recommended based on Craft 3.0 direction)
- Build robust task management to compete with Notion
- Maintain design excellence while adding functionality
- Risk: UI bloat (already a complaint)
- Mitigation: Customizable views, progressive disclosure of features
